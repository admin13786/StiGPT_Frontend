# StiGPT 产品使用文档

## 文档边界

当前项目根目录只保留两个目录：

- `backend`: 后端工程。
- `frontent`: 前端工程、默认 Docker Compose 和项目文档。

启动方式、真实端口、容器编排以 `frontent/README.md` 为准。历史调研报告、旧计划文档和 `_workspace` 目录只作为背景参考，不作为上线入口。

## 当前产品结构

当前用户主线收敛到三组页面：

1. `/apps` 门户首页。
2. `/apps/ai-hub` AI 工作台。
3. `/apps/stigpt/*` 下的 StiGPT 场景入口与 AI 工具页。

## 路由约定

- 登录后的默认入口统一为 `/apps`。
- 对外说明和正式导航统一使用 `/apps/stigpt/*`。
- 旧的 `/stigpt/*`、`/ai/*` 只作为兼容跳转入口。
- 旧的 `support / chat / queue / ticket` 路径不再作为主线页面。

## 核心能力

`/apps/stigpt/webIdx` 是统一聊天入口，支持：

- 新建会话。
- 历史会话切换。
- 流式回复。
- 引用展示。
- 路由模式切换。
- 模型切换。
- 示例问题一键填充。

`/apps/stigpt/write` 是 AI 写作入口，支持：

- 新建写作任务。
- 项目申请和期刊论文两类任务。
- 任务列表和状态查看。
- 资料整理、提纲生成、逐章生成和全文润色。
- 从 AI 检查和 AI 评审回流整改建议。
- 导出 Markdown / TXT。

`/apps/stigpt/check` 是 AI 检查入口，支持：

- 项目、论文、专利三类检查。
- 文档上传。
- 检查报告查看。
- 相似度、风险命中、命中片段和处理建议展示。
- 一键回流到 AI 写作整改。

`/apps/stigpt/review` 是 AI 评审入口，支持：

- 项目评审和论文评审。
- 文档上传。
- 综合评分、维度评分、优势、短板和下一步建议。
- 一键回流到 AI 写作整改。

## 真实服务地址

- 管理端: `http://localhost:32111`
- 玩家端: `http://localhost:32112`
- 后端 API: `http://localhost:32101/api/v1`
- Swagger 管理端: `http://localhost:32101/api/v1/docs/admin`
- Swagger 玩家端: `http://localhost:32101/api/v1/docs/player`

## 上线前检查

上线前至少执行：

```bash
docker compose -f frontent/docker-compose.yml config --quiet
npm --prefix backend run build
npm --prefix frontent/frontend/player-app run build
npm --prefix frontent/frontend/admin-portal run build
```
